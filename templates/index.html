使用边界

先让老师明确当前系统适合处理什么、不适合处理什么。

当前支持

可直接提取文本的 PDF / DOCX，适合命题审查、错字检查和重复题比对。

当前限制

扫描图片卷尚未接入稳定 OCR 主链路，复杂版式仍需关注切题与解析风险提示。

人工复核角色

EchoPaper 负责收敛高风险区，最终确认仍以老师人工判断为准。

当前审查流程

让首页就能说明“上传之后会发生什么”。

  1. 接收教师信息与试卷文件，完成文件类型校验并写入统一处理目录。
  2. 代码版链路优先执行文本提取、规则切题、查重、历史题库比对和本地错字检查。
  3. Agent / Coze 链路异步运行，把本地解析后的题目结构继续送入工作流做对照审查。
  4. 报告页先展示代码版结果，再补充 Agent 任务状态、摘要结果或错误说明。
